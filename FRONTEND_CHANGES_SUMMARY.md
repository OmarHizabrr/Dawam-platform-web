#### أ. `handleTransferLeaves`:
**الوظيفة**: تنفيذ عملية ترحيل الإجازات

### 6. **إضافة Modal للترحيل** ✅

**المكونات**:
- **السنة المصدر**: DatePicker (year picker)
- **السنة الهدف**: DatePicker (year picker) مع التحقق من أن السنة الهدف أكبر من المصدر
- **أنواع الإجازات**: Select متعدد (اختياري)
- **الموظفين**: Select متعدد (اختياري)
- **ملاحظة توضيحية**: توضح أن الرصيد المتبقي فقط سيتم ترحيله

**المميزات**:
- ✅ التحقق من البيانات قبل الإرسال
- ✅ رسائل خطأ واضحة
- ✅ إشعارات نجاح/فشل
- ✅ تحديث تلقائي للبيانات بعد النجاح
- ✅ تصميم متسق مع باقي الواجهة

## 🔗 ربط مع API

### Endpoint:
```
POST /api/transfer-leaves-to-new-year
```

### البيانات المُرسلة:
```javascript
{
  from_year: "2023",           // السنة المصدر
  to_year: "2024",             // السنة الهدف
  vacation_types: [2, 3] || null,  // أنواع الإجازات (اختياري)

}
```

### الاستجابة المتوقعة:

## 🧪 اختبار الواجهة

### الخطوات:
1. فتح صفحة أرصدة الإجازات
2. النقر على زر "ترحيل الإجازات"
3. ملء النموذج:
   - اختيار السنة المصدر (مثلاً: 2023)
   - اختيار السنة الهدف (مثلاً: 2024)
   - اختيار أنواع الإجازات (اختياري)
   - اختيار الموظفين (اختياري)
4. النقر على "ترحيل"
5. التحقق من ظهور إشعار النجاح
6. التحقق من تحديث البيانات في الجدول

## 📝 ملاحظات مهمة

1. **التحقق من البيانات**:
   - السنة الهدف يجب أن تكون أكبر من السنة المصدر
   - جميع الحقول المطلوبة يجب ملؤها

2. **الرسائل**:
   - رسائل نجاح/فشل واضحة
   - عرض عدد السجلات المُرحلة

3. **التحديث التلقائي**:
   - بعد نجاح الترحيل، يتم تحديث الجدول تلقائياً
   - يتم إغلاق Modal وإعادة تعيين النموذج
