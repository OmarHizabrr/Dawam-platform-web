## 🔗 API Endpoint

### المسار:
```
POST /api/transfer-leaves-to-new-year
```

### Request Body:
```json
{
  "from_year": "2023",
  "to_year": "2024",
  "vacation_types": [2, 3],  // اختياري - null = كل الأنواع
  "users": [1, 2, 3]         // اختياري - null = كل الموظفين
}
```

### Response Success:
```json
{
  "success": true,
  "message": "تم الترحيل بنجاح",
  "transferred_count": 15
}
```

### Response Error:
```json
{
  "success": false,
  "message": "يجب تحديد السنة المصدر والهدف"
}
```

## 📝 ملاحظات مهمة

### 1. **الفترة السنوية**:
- النظام يستخدم فترة من **23 ديسمبر للسنة السابقة** إلى **22 ديسمبر للسنة الحالية**
- يجب مراعاة هذا في جميع الحسابات

### 2. **أنواع الإجازات**:
- يتم ترحيل **الإجازات السنوية فقط** (yearly)
- الإجازات الشهرية (monthly) لا يتم ترحيلها

### 3. **الرصيد المتبقي**:
- يتم ترحيل **الرصيد المتبقي فقط** (غير المستهلك)
- الرصيد المتبقي = الممنوح - المستهلك

### 4. **قيمة type في vacations_users**:
- `35` = مرحل من العام الماضي (prev)
- `36` = رصيد العام الحالي (curr)
- `37` = رصيد محول (trans)
